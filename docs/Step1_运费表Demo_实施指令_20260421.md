## 3. 三份文件的结构"坑点备忘"

> **⚠️ 先读这一节再动手。三份文件结构差异极大，`pd.read_excel()` 默认参数全部失效。**

### 3.1 【Air】Market Price

- 工作簿包含 3 个 sheet：
  - `Apr 13 to Apr 19`（上周）
  - `Apr 20 to Apr 26`（本周）
  - `Surcharges`（附加费）
- **周表**（前两个 sheet）：
  - 表头在**第 1 行**
  - 列：`Destinations | Service/+100KG | 2026/4/20(Mon) | 2026/4/21(Tue) | ... | 2026/4/26(Sun) | Remark`
  - 每行一条 `(目的地, 服务/航司/直达或中转)` 记录，7 天 7 列价格
  - Remark 内容示例：`"CK D357"`、`"Must go 22"`、`"BFS TERMINAL"`
- **Surcharges**：
  - **表头在第 4 行，A 列为空**（实际数据从 B 列开始）
  - `F2` 单元格写明 `CURRENCY : CNY`（注意不是表头字段）
  - 字段：`AREA | FROM | AIRLINES | EFFECTIVE DATE | MYC MIN | MYC FEE/kg | MSC MIN | MSC FEE/KG | DESTINATION | REMARKS`
  - `AREA` 和 `FROM` 存在纵向合并单元格（同一 AREA 下多个航司）
  - `EFFECTIVE DATE` 可能是 datetime 也可能是字符串（形如 `"2015-11-01"`），要做宽松解析
  - 金额缺省值用 `-` 表示（不是 0，也不是 null）

### 3.2 【Ocean】Sea Net Rate

- 工作簿包含 3 个 sheet：
  - `JP N RATE FCL & LCL`（日本线 FCL+LCL 混排）
  - `FCL N RATE OF OTHER PORTS`（非日本线 FCL）
  - `LCL N RATE`（LCL 独立表）
- **通用结构**：
  - 1-7 行是元信息（文件号 `HEC-SE-QF-01`、`Effective from B3:D3`、备注等）
  - **表头在第 8 行，数据从第 9 行开始**
  - `Effective from` 解析：`B3` = 起始日期，`D3` = 结束日期
- **JP / FCL 工作表合并单元格陷阱（重要）**：
  - 同一个 `(Destination, Shipping Line)` 组合**占 2 行**：
    - 行 N：第 3 列 = `20FT`，第 4 列 = 20 尺运费
    - 行 N+1：第 3 列 = `40FT/40HQ`，第 4 列 = 40 尺运费
  - `Destination`（A 列）和 `Shipping Line`（B 列）对这 2 行合并
  - **解析时必须按 2 行配对成一条记录**，否则数据错乱
- **列语义**（JP / 其他港 FCL）：
  - USD 段（D-H）：`Freight | LSS+CIC | BAF | EBS | YAS/CAF`
  - 其他（I-K）：`Sailing Day | Via | Transit Time`
  - CNY 段（L-P）：`Booking Charge | THC(per Cont) | DOC(per BL) | ISPS | Equipment management`
  - Q 列：`RMKS`
- **LCL 工作表特殊**：
  - Freight 字段形如 `"5/CBM, 10/TON"` —— **双单位混合**，必须拆成 `freight_per_cbm` + `freight_per_ton`
  - 附加费列用 `Included` / `-` 表示，不是数字

### 3.3 【Ocean-NGB】

- 工作簿包含 3 个 sheet：
  - `sample`（模板示例，**跳过不导**）
  - `Rate`（真实费率数据，**只处理这个**）
  - `Shipping line name`（船司字典，可作为 lookup，不入主表）
- **Rate 表结构**：
  - 表头在**第 1 行**，数据从第 2 行
  - 共 55 列
  - `P/S rate or N` 列（B 列）的值：`Lv.1 / Lv.2 / Lv.3 / Net/Net / P/S rate`
    - **同一条航线有 Lv.1/Lv.2/Lv.3 三行**，必须作为不同等级保留
  - 容器列命名：`20' GP per CNTR` / `40' GP per CNTR` / `40'HC per CNTR`
  - `Rate Valid from` / `Rate Valid until` 是 datetime

## 5. 解析器架构

每份文件一个独立 Adapter，实现统一接口：

```python
class RateAdapter(Protocol):
    file_type: str  # "air" | "ocean" | "ocean_ngb"

    def detect(self, path: Path) -> bool:
        """按文件名 / sheet 名指纹识别是否本类型"""

    def parse(self, path: Path) -> ParsedRateBatch:
        """返回统一中间结构：batch 元信息 + 多个 rate records"""
```

实现类：

- `AirAdapter`：处理周表 + Surcharges 两类输出
- `OceanAdapter`：处理 JP / Other Ports / LCL 三个 sheet，**FCL 必须按 2 行配对**
- `OceanNGBAdapter`：只读 `Rate` sheet，保留 Lv 分级

**统一中间结构 `ParsedRateBatch`**：

```python
@dataclass
class ParsedRateBatch:
    file_type: str
    source_file: str
    effective_from: date | None
    effective_to: date | None
    records: list[ParsedRateRecord]
    warnings: list[str]  # 解析过程中的疑似问题
```

## 6. 导入更新机制（半月更新必备）

- **批次键**：`(file_type, effective_from, effective_to)` 组合
- **入库流程**：
  1. 上传 → 解析 → 生成 `import_batch`（`status=draft`） + 中间记录落 staging
  2. 前端调用差异接口 → 对比 staging vs 当前 `active` 批次
  3. 返回 `{新增: N 条, 变更: M 条, 消失: K 条, 细节列表}`
  4. 用户确认 → 新批次置 `active`，旧同键批次置 `superseded`（**不物理删除**）
- **原始文件归档**：`data/tariffs/raw/<batch_id>/<filename>` —— 永久留底

## 7. 运费表输出（原格式回填）

**核心原则：用原文件作模板，只改数据单元格，不破坏结构。**

实现方式：

```python
from openpyxl import load_workbook
wb = load_workbook("templates/ocean_template.xlsx")  # 原件剥离数据后的空壳模板
# 逐单元格写入，保留合并、样式、附注
```

**禁止**使用 `pandas.to_excel()` 或 `DataFrame.to_excel()` 重新写——会丢失所有样式和合并。

**模板准备**：
- 基于三份原件，手工清空数据行、保留元信息/表头/附注/样式，存 `data/templates/`
- Ocean FCL 回填时，按"20FT 行 + 40FT/40HQ 行"配对写入

## 0. 真实 Excel 采样结果（与业务文档的偏差，必须先看）

采样文件：`资料/2026.04.21/RE_ 今後の進め方に関するご提案/【Air】 Market Price updated on  Apr 20.xlsx`，采样日期 2026-04-22，用 `openpyxl.load_workbook(..., data_only=True)`。

### 0.1 工作簿确认

- `wb.sheetnames = ['Apr 13 to Apr 19', 'Apr 20 to Apr 26', 'Surcharges']`
- 两张周表 `max_row=46, max_col=14`，`Surcharges` `max_row=70, max_col=11`
- 周表表头在**第 1 行**，数据第 2–43 行，第 44 行起全空
- Surcharges 表头在**第 4 行**，数据第 5–67 行，A 列整列空

### 0.2 与业务文档 §4.2 的关键偏差（业务大师说法不准确，必须以实测为准）

- **业务文档说**："`AREA` 和 `FROM` 存在纵向合并单元格"
- **实测**：
  - B 列（AREA）**不是合并单元格**。只有 `B5='TC-3'`，`B6..B67=None`。必须用"向下填充"策略（forward-fill），不能用 `ws.merged_cells` 展开。
  - C 列（FROM）才是真正的合并单元格 `C5:C67`，值为 `'CHINA / SHA \n\n\n\n…注\n意\n生\n效\n日\n期'`（内含竖排"注意生效日期"汉字）。
  - 合并单元格清单仅一个：`C5:C67`（Surcharges）；周表 Remark 列在 `J{n}:K{n}` 逐行合并，共 42 条。
- **结论**：解析时 AREA 用 forward-fill，FROM 用 `ws.merged_cells` 展开或直接对合并区域赋常量。两种机制**不可混用**。

### 0.3 周表 Remark 列位置

- 表头 `'Remark (Selling)'` 在 `J1`，与 `K1` 合并（J1:K1）
- 每个数据行 Remark 也在 `J{n}:K{n}` 合并，读 J 列即可（K 列永远是 None）
- 第 10 列 = J 列 = Remark；第 11 列 = K 列（只做显示宽度用）；第 12–14 列（L/M/N）整列为空

### 0.4 Service 列原文瑕疵（必须原样保留，不得纠错）

- R3 `'HO 2 days servcie'`（servcie 拼写错误）
- R22 `'MH/FM 3 days servcie '`（拼写错误 + 末尾空格）
- R27 `"BR/SQ 3-4 days\nservice"`（含换行符）
- R32/R34/R35/R36 服务列也含换行
- R41 `'FRA VOLUME '`（目的地末尾空格）
- 业务规则：`service_desc` 存**strip 后原文**（去首尾空白，保留内部换行），`extras.raw_service`存**未处理原文**

### 0.5 真空值 vs `-` 的共存（实证）

- `F5='-'`（AA 行 MYC MIN 不征收保底）
- `F66=None, H66=None`（Y8 TC-1 MYC MIN/MSC Min 字段真空）
- 两种语义需分别标识，不可归一化为同一值

### 0.6 EFFECTIVE DATE 类型确认

- 实测本文件所有 `E5..E67` 值都是 `datetime.datetime`，没有字符串
- 但 §3.1 实施指令提示"可能是字符串"，**解析器必须宽松处理**（`datetime`/`date`/字符串均支持）

## 2. Sheet 识别策略

### 2.1 周表识别

- **规则**：sheet 名匹配正则 `^[A-Z][a-z]{2}\s+\d{1,2}\s+to\s+[A-Z][a-z]{2}\s+\d{1,2}$`
- **伪代码**：
  ```
  _WEEKLY_SHEET_RE = re.compile(r"^([A-Z][a-z]{2})\s+(\d{1,2})\s+to\s+([A-Z][a-z]{2})\s+(\d{1,2})$")
  ```
- **为何不用"第二个 sheet"位置法**：客户可能追加历史周表；位置法在 3 sheet 以上时失效。
- **兜底**：若 0 个周表被识别 → warning `"no weekly sheet matched pattern 'Mon dd to Mon dd'"`，adapter 继续跑 Surcharges。

### 2.2 Surcharges 识别

- **规则**：`sheet_name.strip().lower() == "surcharges"`
- **兜底**：若缺失 → warning `"Surcharges sheet missing, skipped airline surcharge parsing"`。

### 2.3 两张周表的处置（Q1 技术兜底）

- 业务文档 §5.3 有歧义，本方案**两张周表全部入库，用 `record_kind` 和 `effective_week_start` 区分**：
  - 当前周：`record_kind='air_weekly'`（主）
  - 历史周：`record_kind='air_weekly'`（同类，靠 `effective_week_start` 区分）
- 不在 adapter 层做"丢弃上周"判断——adapter 只负责产出所有可解析记录；"本周 vs 上周"的去重/active 标识交给批次服务层（批次键 `(file_type, effective_from, effective_to)` = 本周最大日期范围）。
- 本 adapter 的 `ParsedRateBatch.effective_from/effective_to` 取**当前周**（工作簿中 `effective_week_start` 最大的那张 sheet）。若无法判定，取最后一个识别到的周表。
- **附带 warning**：如工作簿含 ≥2 张周表，发 warning `"workbook contains N weekly sheets: [...]; batch effective range uses the latest week {start} - {end}"`。

## 3. 周表解析算法

### 3.1 数据区扫描范围

- 从第 2 行开始到第 43 行（含）；实际取 `row in range(2, min(ws.max_row, 100) + 1)`，遇到"Destinations 和 Service 同时为空"连续出现 ≥2 次即提前终止。
- 不硬编码 43，给未来扩行留余地。

### 3.2 列映射（固定列，不做表头动态识别）

| Excel 列 | 索引（1-based） | 字段 |
|---|---|---|
| A `Destinations` | 1 | 目的地原文 |
| B `Service/+100KG` | 2 | Service 原文 |
| C-I | 3–9 | 7 日日报价，依次映射 price_day1..price_day7 |
| J `Remark (Selling)` | 10 | Remark 原文（K 列跳过） |

- **不动态识别表头**：周表结构 2 周都一致，实施指令 §3.1 已冻结列顺序；若 `ws.cell(1,1).value != "Destinations"` → warning + 继续按固定列解析。

### 3.3 一行一记录

- 每个非空数据行产出**一条** `ParsedRateRecord`，`record_kind='air_weekly'`
- 不按目的地合并；BKK 的 4 条各自独立（业务文档 §3.1、§5.1）

### 3.4 price_day1..price_day7 规则

- 逐列读 C..I（第 3..9 列）
- 值为数字 → `Decimal(str(value))`
- 值为 None / 空字符串 → 对应字段 `None`，`extras.price_day_missing.append(<1..7>)`，并发 warning `"{sheet} row {r}: price_day{n} is empty"`（Q3 技术兜底：**不自动用前一日填充**，只如实记录，留给业务侧判断）
- 值为非数字字符串 → warning `"{sheet} row {r}: price_day{n} contains non-numeric '{raw}'"`，字段 None，`extras.price_day{n}_raw` 存原文

### 3.5 Service 列 → service_desc（不拆分）

- `service_desc = _normalize_text(B 列)`（strip、内部空白折叠为单空格；**保留换行 `\n`** → 改用 `re.sub(r"[ \t]+", " ", text).strip()`）
- 不拆分航司/天数/直达中转；业务文档 §3.3 明确半结构化（Q6 技术兜底）
- `extras.raw_service` 存未处理原文（含末尾空格）

### 3.6 extras.airline_codes（辅助字段，不入主字段）

- 从 `service_desc` 抽"疑似航司二字代码"：正则 `\b[A-Z]{2}\b`（要求前后是非字母边界或行首行尾）
- 进一步筛：去掉英文通用词如 `TO/ON/IN/AT/BY/NH`（白名单冲突，NH 是航司代码 — **不过滤白名单**，直接正则抽取）
- 伪代码：
  ```
  codes = re.findall(r"(?<![A-Za-z])[A-Z]{2}(?![A-Za-z])", service_desc)
  extras["airline_codes"] = sorted(set(codes))
  ```
- 主字段 `airline_code` **留 None**（原因：`CK/MU`、`XJ/CI/SQ/NH` 等多航司串，无法确定单一值；强行选一个会误导营业）
- Q6 兜底：未来 Step2 检索时从 `extras.airline_codes` 做 `in` 过滤

### 3.7 Remark 列原文保留 + 布尔提示

- `remarks = _normalize_text(J 列)`，保留换行
- `extras.raw_remark` 存未处理原文
- 布尔提示写入 extras，不入主字段：
  - `extras.has_must_go = "must go" in remark_lower`
  - `extras.must_go_value = <从 "Must go 22" 抽出的数字>`（regex `r"must\s*go\s*([0-9]+(?:\.[0-9]+)?)"`）
  - `extras.is_case_by_case = "case by case" in remark_lower`
- Q4/Q5 兜底：原文永久保留；must_go/case_by_case 只做提示，具体语义由 Step2 决定

### 3.8 目的地特殊形态（核心业务坑）

业务文档 §9 明确警告此处最易踩坑。

| 原文形态 | 示例行 | `destination_port_name` 主字段 | extras 辅助 |
|---|---|---|---|
| 单机场 IATA | R2 `NRT` | `"NRT"` | `extras.destination_airports=["NRT"]` |
| 多机场斜杠 | R33 `DXB/DWC` | `"DXB/DWC"`（保留原文） | `extras.destination_airports=["DXB","DWC"]` |
| 机场+密度 | R40 `FRA DENSE` / R41 `FRA VOLUME ` | `"FRA DENSE"` / `"FRA VOLUME"`（strip） | `extras.destination_airports=["FRA"]`，`extras.density_hint="DENSE"` 或 `"VOLUME"` |

- 主字段永远保留原文 strip 后的值
- `extras.destination_airports` 抽取规则：按 `/` 和空白切，筛长度=3 且全大写字母的片段
- `extras.raw_destination` 存未处理原文
- **不强转 port_id**：`origin_port_id/destination_port_id` 对 Air 周表**固定填 None**（数据库 `ports` 表是海港为主；Air 机场代码独立命名空间，本轮不引入机场代码表 — Q10 兜底）

### 3.9 origin 规则

- Air 周表**没有 origin 字段**；业务默认起运地 = 上海浦东（购买部门在上海）
- `origin_port_id = None`
- `origin_port_name = "PVG"`（硬编码，加 `extras.origin_source="default_air_PVG"`）
- 加一条 metadata `"air_origin_assumption": "PVG (no origin column in weekly sheet)"`（batch 级别，只发一次，避免 42 条 warning 刷屏）

### 3.10 effective_week_start / effective_week_end

- 从 sheet 名 `Apr 20 to Apr 26` 解析；年份从**同行 C1 单元格**的日期提取（C1 值为 `'2026/4/20   (Mon)'`）
  - 先尝试 `datetime.datetime.strptime("2026/4/20", "%Y/%m/%d")` 正则抽出；失败退化到 sheet 名 + 当前年份
- 伪代码：
  ```
  # 优先从 C1/I1 读年份
  year = _extract_year_from_header(ws["C1"].value) or date.today().year
  match = _WEEKLY_SHEET_RE.match(sheet_name)
  start = date(year, _month_abbr_to_num(match[1]), int(match[2]))
  end   = date(year, _month_abbr_to_num(match[3]), int(match[4]))
  if end < start:  # 跨年
      end = end.replace(year=year + 1)
  ```
- 每条记录的 `effective_week_start/effective_week_end` 都写入；`valid_from/valid_to` 同步写（复用现有字段）
- 批次级 `effective_from/effective_to` = 当前周（见 §2.3）

### 3.11 currency（Q2 技术兜底）

- 周表无币种声明
- 本 adapter 固定 `currency="CNY"`（推断依据：Surcharges 明示 CNY、周表数字量级 5–55 符合 CNY/kg 典型值、上海购买部门对内表）
- `extras.currency_assumption="inferred_CNY_no_header"` 每条都带（batch metadata 里也发一条 warning `"weekly sheet currency not declared; inferred as CNY"`）
- 若 Q2 业务回复为 USD，开发大师只需改一处常量

### 3.12 周表字段映射完整表

| sheet 名 | `effective_week_start`, `effective_week_end`, `valid_from`, `valid_to` | 见 §3.10 |

| A 列 strip | `destination_port_name` | §3.8 |

## 4. Surcharges 解析算法

### 4.1 数据区扫描范围

- 从第 5 行到第 67 行（含）；实际 `row in range(5, min(ws.max_row, 100) + 1)`
- 遇到 D 列（AIRLINES）为空的行跳过（向下扫，不提前 break，因为 R68–R70 确实全空但格式化区可能留行）
- 连续 ≥3 行 D 列为空 → break

### 4.2 表头校验（第 4 行）

- 读第 4 行 B4..K4，与期望值对比：`["AREA","FROM","AIRLINES","EFFECTIVE DATE","MYC MIN","MYC FEE/kg","MSC Min","MSC FEE/KG","DESTINATION","REMARKS"]`
- 允许大小写/空白差异（`_normalize_text + .lower()` 后比较 key set）
- 不匹配 → warning `"Surcharges header mismatch at row 4; expected AREA/FROM/..., got [...]"`，**继续按固定列解析**（不 fail-fast）

### 4.3 币种读取（F2）

- `currency_raw = ws["F2"].value`（实测 `'CURRENCY : CNY'`）
- 正则抽取：`re.search(r"CURRENCY\s*:\s*([A-Z]{3})", currency_raw, re.IGNORECASE)`
- 命中 → 本 sheet 所有记录 `currency=<抽出值>`
- 未命中 / F2 为空 → fallback `currency="CNY"` + warning `"Surcharges F2 currency tag missing or unparseable, fallback to CNY"`

### 4.4 AREA 和 FROM 处理（与业务文档 §4.2 偏差）

- **AREA（B 列）**：forward-fill
  - 维护 `last_area = None`，每行若 B 列有值则更新 `last_area`，否则用 `last_area`
  - 实测 B5='TC-3'，其余 None → 全部填 'TC-3'
- **FROM（C 列）**：`ws.merged_cells` 展开
  - 本文件 C5:C67 是一个合并单元格，值 `'CHINA / SHA \n\n\n…注\n意\n生\n效\n日\n期'`
  - 读取后清洗：保留第一行 `'CHINA / SHA'`（正则 `re.split(r"\n|\s{2,}", value)[0].strip()`）
  - 原始值存 `extras.raw_from`（含"注意生效日期"竖排汉字）
- Q7 兜底：未来若出现第二个 AREA/FROM，forward-fill 和 merged_cells 逻辑天然支持

### 4.5 数据列映射

| Excel 列 | 索引 | ParsedRateRecord 字段（或 extras） |
|---|---|---|
| B AREA | 2 | `extras.area` |
| C FROM（清洗后） | 3 | `extras.from_region` |
| D AIRLINES | 4 | `airline_code`（原文 strip；如 `'AA - American Airlines'`） + `extras.airline_code_raw` + `extras.airline_iata`（从 D 列 `/` 或 `-` 前的前缀抽，如 `"AA"`、`"MU"`、复合如 `"MU/CK"`） |
| E EFFECTIVE DATE | 5 | `valid_from`（date） + `extras.effective_date_raw` |
| F MYC MIN | 6 | `extras.myc_min_value`（Decimal or None）+ `extras.myc_min_is_dash`（bool）|
| G MYC FEE/kg | 7 | `extras.myc_fee_per_kg` + `extras.myc_fee_is_dash` |
| H MSC Min | 8 | `extras.msc_min_value` + `extras.msc_min_is_dash` |
| I MSC FEE/KG | 9 | `extras.msc_fee_per_kg` + `extras.msc_fee_is_dash` |
| J DESTINATION | 10 | `extras.destination_scope`（原文 strip） |
| K REMARKS | 11 | `remarks` |

**主字段占用说明**：
- `record_kind = "air_surcharge"`
- `origin_port_id/origin_port_name = None / "PVG"`（统一约定）
- `destination_port_id/destination_port_name = None`
- `service_desc = None`
- `price_day1..price_day7 = None`
- `effective_week_start/effective_week_end = None`
- `carrier_name = <D 列原文 strip>`（便于检索页最小可用维度）
- `airline_code = <D 列 `-` 或 `/` 前的前缀抽取结果>`；多航司组合（如 `MU/CK`）保留 `/` 原样

### 4.6 `-` vs 真空值的三态处理（Q11/Q12 兜底）

- `_to_decimal_or_dash(value)` 返回 `("value", Decimal)` / `("dash", None)` / `("empty", None)` 三态
- 对应 extras 写两个字段：数值字段（`*_value`）+ 标志字段（`*_is_dash`）
- **语义约定**（不等 Q11/Q12 业务回复，先落）：
  - `-` → `value=None, is_dash=True` → 表示"明示不征收"
  - 真空 → `value=None, is_dash=False` → 表示"未填/待确认"
- `extras.all_fees_dash = True`（当 F/G/H/I 四字段全 `-` 时）→ 让检索层可过滤掉 BA 这类条目
- `extras.all_fees_empty = True`（四字段全真空）→ warning `"Surcharges row {r} ({airline}): all four fee fields empty"`

### 4.7 EFFECTIVE DATE 宽松解析

- 支持 `datetime.datetime / datetime.date / str`
- `_to_date` 尝试顺序：`isinstance(datetime)` → `.date()`；`isinstance(date)` → 直接用；`str` → `datetime.strptime` 试 `"%Y-%m-%d"` / `"%Y/%m/%d"` / `"%Y.%m.%d"`；失败 → None + warning
- Q9 兜底：只管落日期，覆盖/留档由批次层决定

## 7. detect() 如何区分 Air / Ocean / Ocean-NGB

现状见 `air.py:17-21` / `ocean.py:41-45` / `ocean_ngb.py:17-21`。

本轮 Air detect **微调**（不改 priority，保持 10）：

```python
def detect(self, path, *, file_type_hint=None):
    if file_type_hint == self.file_type:
        return True
    name = path.name.lower()
    # 排除 ocean-ngb / ocean，防止"ocean"子串误命中 air 关键词
    if "ocean" in name or "ngb" in name:
        return False
    return "air" in name or "market price" in name
```

- priority 10 < ocean 20 < ocean_ngb 30，Air 先匹配，但含 `ocean`/`ngb` 的直接让给后者
- 与 ocean.py:41-45 已有的 `"ocean" in name and "ngb" not in name` 互斥
- 验收点：三份真实文件用 `registry.resolve()` 各自命中对应 adapter（见 §10 验收）

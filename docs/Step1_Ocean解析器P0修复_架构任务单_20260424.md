## 任务 T-B：消除 LCL freight 列 'MBL CC' 噪音 warning

- **对应业务需求**：P0-C 遗留（166 warnings 中 4 条 `'MBL CC'` 噪音误报）
- **改动范围**：
  - 修改文件：`backend/app/services/step1_rates/adapters/ocean.py`
    - **改动点 1（新增模块级常量）**：在 L38 `_KNOWN_NON_NUMERIC_MARKERS` 元组末尾闭合 `)` 之后、L40 `def detect` 之前插入：
      ```python
      _LCL_FREIGHT_PLACEHOLDERS = frozenset({"-", "mbl cc"})
      ```
    - **改动点 2（修改方法体）**：修改 `_parse_lcl_freight` L687-717，在 L692-693 现有的 `if not freight_raw` 空值短路之后、L695 `normalized = freight_raw.replace(" ", "").upper()` 之前，新增一段占位符短路：若 `freight_raw.strip().lower()` 命中 `_LCL_FREIGHT_PLACEHOLDERS`，记录 `extras["freight_parse_status"] = "placeholder"`，直接 `return None, None, extras, None`（warning 位返回 None，即不产生 warning）
- **接口契约**（严格照当前实际签名，不采纳概要里的 3 元组误写）：
  ```
  def _parse_lcl_freight(
      self,
      freight_raw: str | None,
  ) -> tuple[Decimal | None, Decimal | None, dict[str, Any], str | None]:
  ```
  签名不变；返回顺序 `(per_cbm, per_ton, extras, warning)` 不变；仅行为变化：命中占位符时 warning 位返回 None。
- **数据流**：
  - 输入：LCL sheet B 列 `freight_raw = 'MBL CC'`（共 4 条）
  - 匹配：`freight_raw.strip().lower() == 'mbl cc'` ∈ `_LCL_FREIGHT_PLACEHOLDERS`
  - 中间态：`extras["freight_parse_status"] = "placeholder"`
  - 输出：`(None, None, extras, None)`
  - 上游 `_parse_lcl_sheet` L296-300：`freight_warning is None` → 不 append 到 warnings
  - 上游 L309 `extras["freight_raw"] = freight_raw` 仍保留原文 `'MBL CC'`（不丢信息）
- **依赖**：无（不依赖 T-A / T-C）
- **迁移步骤**：纯解析器改动，不涉及数据库，无 Alembic；上线后重跑已归档批次不会产生存量差异（历史批次的 warnings 列表不回溯修改）
- **验收点**（交给测试大师）：
  1. LCL sheet 跑完，`warnings` 列表中 **不再包含** `"non-standard freight value 'MBL CC' kept as raw text"` 条目（当前应为 4 条，修复后应为 0 条）
  2. 受影响行的 `extras["freight_raw"]` 仍为原文 `'MBL CC'`（不丢信息）
  3. 受影响行的 `extras["freight_parse_status"]` 为 `"placeholder"`
  4. 受影响行的 `freight_per_cbm` / `freight_per_ton` 均为 `None`
  5. 新增一个单元测试文件 `backend/tests/services/step1_rates/adapters/test_ocean_lcl_freight.py`，直接调用 `OceanAdapter()._parse_lcl_freight('MBL CC')` 与 `OceanAdapter()._parse_lcl_freight('-')`，断言均返回 `(None, None, {"freight_parse_status": "placeholder"}, None)`
- **风险**：
  - 若将来客户在 B 列放其它占位符（如 `"PENDING"`、`"TBD"`、`"N/A"`），需要扩 `_LCL_FREIGHT_PLACEHOLDERS` 集合。**开发大师不得擅自扩充**，任何新增占位符需走"业务大师提需求 → 架构大师回炉"的流程
  - `frozenset` 比较用的是 lowercase，若客户后续大小写异常（例如 `'mbl Cc'`、`'MBL  CC'`），`freight_raw.strip().lower()` + 空白归一需配合。当前 `strip().lower()` 只处理首尾空白 + 大小写，不处理中间多空白。若实际数据出现中间双空格 `'MBL  CC'`，命中会失败 — 若监控到此类数据，走回炉流程补 `_WHITESPACE_RE.sub` 归一

## 任务 T-C：LCL freight `0/RT` `0/CBM` `0/TON` 零值识别

- **对应业务需求**：P0-C 遗留 — BUSAN 行 `0/RT` 等零值若被当作合法费率存库，Demo 现场会被楢崎当场发现
- **改动范围**：
  - 修改文件：`backend/app/services/step1_rates/adapters/ocean.py`
    - **改动点**：修改 `_parse_lcl_freight` L687-717 的四个分支出口，每个分支在返回前加一个零值判定：
      - L696-699（`/CBM` 或 `/TON` 组合分支）：若 `per_cbm` 或 `per_ton` 解析出的 Decimal 值为 `0`，对应字段置 None 并追加 warning `"zero freight rate ignored: {freight_raw}"`
      - L701-704（`/RT` 分支）：若 `value == Decimal('0')`，返回 `(None, None, extras, f"zero freight rate ignored: {freight_raw}")`
      - L706-709（`/CBM` 单独分支）：同上
      - L711-714（`/TON` 单独分支）：同上
  - 新增测试：追加到 `backend/tests/services/step1_rates/adapters/test_ocean_lcl_freight.py`（与 T-B 共用）
- **接口契约**：签名不变（仍为 4 元组 `(per_cbm, per_ton, extras, warning)`）；行为变化：零值不再写入 `per_cbm` / `per_ton`，warning 位返回提示文本
- **数据流**：
  - 输入：LCL `freight_raw = '0/RT'`（BUSAN 行，Row 14）
  - 旧行为：`_safe_decimal('0')` → `Decimal('0')`，返回 `(Decimal('0'), Decimal('0'), {"freight_unit": "RT"}, None)`
  - 新行为：解析出 `Decimal('0')` → 判零 → 返回 `(None, None, {"freight_unit": "RT"}, "zero freight rate ignored: 0/RT")`
  - 上游：`_parse_lcl_sheet` L296-300 将 warning append 到 warnings；L309 `extras["freight_raw"]` 仍保留 `'0/RT'`
- **依赖**：**T-B**（建议先合 T-B 再合 T-C）。理由：T-B 在函数入口加的是 placeholder 短路，T-C 改的是后续各分支出口。若 T-C 先进，T-B 再在入口插入短路，合并时需确认 `_LCL_FREIGHT_PLACEHOLDERS` 短路不会截断 T-C 的零值路径（当前 `'0/RT'` 不在 placeholder 集合里，不会被截断，但代码审查时需明确标注顺序）
- **迁移步骤**：同 T-B，纯解析器改动，不涉及数据库，无 Alembic
- **验收点**（交给测试大师）：
  1. **BUSAN Row 14 `0/RT`**：解析后 `freight_all_in` / `freight_per_cbm` / `freight_per_ton` 均为 `None`
  2. **extras 保真**：`extras["freight_raw"] == '0/RT'` 保留；`extras["freight_unit"] == 'RT'` 保留
  3. **warning 追加**：`warnings` 列表中包含 `"zero freight rate ignored: 0/RT"` 一条（或等价文案，文案需与实现一致）
  4. **`0/CBM` 同规则**：`_parse_lcl_freight('0/CBM')` → `(None, None, {"freight_unit": "CBM"}, "zero freight rate ignored: 0/CBM")`
  5. **`0/TON` 同规则**：`_parse_lcl_freight('0/TON')` → `(None, None, {"freight_unit": "TON"}, "zero freight rate ignored: 0/TON")`
  6. **组合分支零值**：`_parse_lcl_freight('0/CBM, 0/TON')` → `per_cbm=None, per_ton=None`，warning 包含零值提示（具体文案由开发大师实现时决定，测试断言子串 `"zero freight rate ignored"`）
- **风险**：
  - 若客户**有意用 0 表示"免费"**，此改动会误杀。当前判断：0 在运费上下文中不是合法语义，若客户 Demo 后抗议再恢复
  - `_safe_decimal` 对 `'0.00'`、`'0.0'` 的返回均为 `Decimal('0')`，需用 `== Decimal('0')` 或 `.is_zero()` 判断，不得用 `not value`（会把 `None` 误判为零）

## 监工勘误单（T-B 验收点 2）

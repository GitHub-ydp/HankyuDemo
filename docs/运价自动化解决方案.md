# 阪急阪神 运价自动化解决方案

## 一、方案概述

### 1.1 目标

针对阪急阪神购买部门运价管理的核心痛点，构建 **AI 驱动的运价自动解析与管理系统**，实现：

- **多格式运价自动解析**：Excel、PDF、邮件文本、WeChat 截图 → 统一结构化数据
- **统一费率数据库**：取代分散的 Excel 管理，实现集中存储、实时查询
- **投标包自动填写**：AI 自动匹配费率并填入客户投标包模板
- **人工审核流程**：所有 AI 处理结果经人工确认后方可使用

### 1.2 核心价值

| 指标 | 现状 | 方案目标 |
|------|------|---------|
| 运价录入 | 人工逐条抄写，每次数小时 | AI 自动解析，上传即入库 |
| 数据管理 | 分散在 Excel 中，查找困难 | 统一数据库，按航线/船司/有效期秒级查询 |
| 投标填写 | 逐航线手动填入，容易出错 | AI 自动匹配+填写，人工审核确认 |
| 更新频率 | 约2周更新一次 | 收到即更新，数据始终最新 |

## 三、核心功能模块

### 3.1 多格式运价解析引擎

这是系统的核心，针对4种数据源各有专门的解析策略：

#### 3.1.1 Excel 费率表解析（parse_excel_tariff）

**处理对象**：KMTC 运价表、NVO FAK 等结构化 Excel

**技术方案**：
```
上传 Excel → openpyxl 读取 → 自动识别表头行和列结构
→ 映射到标准化 Schema → 数据清洗（公式求值、空值处理）
→ 批量入库
```

**关键难点与解决**：
- **列结构差异**：不同供应商 Excel 格式不同 → 基于表头关键词自动匹配（如含"20GP"/"20ft"/"20'"的列映射到 container_20gp）
- **公式引用**：部分单元格是 `=F6` 这样的公式 → openpyxl `data_only=True` 模式读取计算值
- **编码问题**：中文港口名乱码 → 统一 UTF-8 处理

#### 3.1.2 PDF 合约解析（parse_pdf_contract）

**处理对象**：ONE Service Contract 等船司 PDF 合约（可达数百页）

**技术方案**：
```
上传 PDF → pdfplumber 逐页提取文本 → 定位费率表页面
→ 提取表格结构 → Claude 辅助结构化（处理复杂布局）
→ 标准化 → 入库
```

**关键难点与解决**：
- **页数多**：372页需高效定位 → 先扫描关键词（"RATE"/"USD"/"ORIGIN"）定位费率页范围
- **表格跨页**：费率表可能跨多页 → 连续页面合并处理
- **异步处理**：大文件通过 Celery 后台处理，前端显示进度

#### 3.1.3 邮件文本解析（parse_email_text）

**处理对象**：邮件正文中的纯文本报价（如长荣市场报价）

**技术方案**：
```
粘贴/上传文本 → Claude NLP 实体提取
→ 识别：船司名、航线代码、目的港、柜型费率、有效期
→ 结构化输出 → 人工确认 → 入库
```

**Claude Prompt 设计要点**：
- 提供物流行业术语上下文
- 明确输出 JSON Schema
- 处理中文口语化表达（"胡志明卡莱"→"Ho Chi Minh / Cat Lai"）
- 处理费率缩写格式（"375/750"→20GP:375, 40GP:750）

#### 3.1.4 WeChat 截图解析（parse_wechat_image）

**处理对象**：WeChat/QQ 聊天中的运价截图

**技术方案**：
```
上传图片 → Claude Vision 多模态识别
→ 提取对话中的运价信息（港口、费率、有效期、备注）
→ 结构化输出 → 人工确认 → 入库
```

**关键难点与解决**：
- **对话格式**：非表格格式，信息分散在对话气泡中 → Claude Vision 理解对话上下文
- **数字精度**：运价数字必须准确 → 提取后展示给用户确认

### 3.2 统一费率数据库

#### 3.2.1 数据模型

```sql
-- 港口字典表
CREATE TABLE ports (
    id SERIAL PRIMARY KEY,
    un_locode VARCHAR(10) UNIQUE,     -- UN/LOCODE (如 CNSHA)
    name_en VARCHAR(100),              -- 英文名
    name_cn VARCHAR(100),              -- 中文名
    country VARCHAR(50),               -- 国家
    region VARCHAR(50)                 -- 区域（东南亚/韩国/美洲等）
);

-- 船司/供应商表
CREATE TABLE carriers (
    id SERIAL PRIMARY KEY,
    code VARCHAR(20) UNIQUE,           -- 船司代码 (KMTC, ONE, EMC)
    name_en VARCHAR(100),
    name_cn VARCHAR(100),
    carrier_type VARCHAR(20)           -- shipping_line / co_loader / agent
);

-- 费率主表
CREATE TABLE freight_rates (
    id SERIAL PRIMARY KEY,
    carrier_id INTEGER REFERENCES carriers(id),
    origin_port_id INTEGER REFERENCES ports(id),
    destination_port_id INTEGER REFERENCES ports(id),
    service_code VARCHAR(20),          -- 航线代码
    container_20gp DECIMAL(10,2),      -- 20尺柜运费 USD
    container_40gp DECIMAL(10,2),
    container_40hq DECIMAL(10,2),
    container_45 DECIMAL(10,2),
    baf_20 DECIMAL(10,2),              -- 燃油附加费
    baf_40 DECIMAL(10,2),
    lss_20 DECIMAL(10,2),              -- 低硫附加费
    lss_40 DECIMAL(10,2),
    currency VARCHAR(5) DEFAULT 'USD',
    valid_from DATE,
    valid_to DATE,
    transit_days INTEGER,
    is_direct BOOLEAN DEFAULT TRUE,    -- 直达/中转
    remarks TEXT,
    source_type VARCHAR(20),           -- excel/pdf/email_text/wechat_image
    source_file VARCHAR(255),
    upload_batch_id VARCHAR(50),       -- 同一次上传的批次ID
    status VARCHAR(20) DEFAULT 'active', -- active/expired/draft
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 上传记录表
CREATE TABLE upload_logs (
    id SERIAL PRIMARY KEY,
    batch_id VARCHAR(50) UNIQUE,
    file_name VARCHAR(255),
    file_type VARCHAR(20),
    source_type VARCHAR(20),
    records_parsed INTEGER,            -- 解析出的记录数
    records_imported INTEGER,           -- 成功入库的记录数
    status VARCHAR(20),                -- processing/completed/failed
    error_message TEXT,
    uploaded_by VARCHAR(100),
    created_at TIMESTAMP DEFAULT NOW()
);
```

#### 3.2.2 查询接口

| 接口 | 方法 | 功能 |
|------|------|------|
| `GET /api/rates` | GET | 按条件查询费率（起运港/目的港/船司/有效期/柜型） |
| `GET /api/rates/compare` | GET | 同航线多供应商比价 |
| `GET /api/rates/export` | GET | 导出查询结果为 Excel |
| `GET /api/ports` | GET | 港口字典查询 |
| `GET /api/carriers` | GET | 船司/供应商列表 |

### 3.3 投标包自动填写（fill_pkg）

**处理流程**：
```
上传客户 PKG 模板 → AI 解析模板结构（识别航线栏位、费率栏位）
→ 从费率数据库匹配对应费率 → 自动填入 → 生成预览
→ 人工审核/修改 → 下载完成的 PKG 文件
```

> 注：此功能在 Demo 阶段作为展示，待获取客户真实 PKG 模板后深入开发。

### 3.4 前端界面

| 页面 | 功能 |
|------|------|
| **文件上传** | 拖拽上传多种格式文件，自动识别类型，显示解析进度 |
| **解析预览** | 展示 AI 解析结果表格，高亮异常值，支持编辑修改、确认入库 |
| **费率管理** | 全量费率数据浏览、多条件筛选、排序、导出 |
| **航线比价** | 输入起运港+目的港，展示所有供应商报价对比 |
| **上传记录** | 历史上传记录、解析统计、错误日志 |

## 四、Demo 实施计划

### 4.1 Demo 目标

用福山提供的 **真实数据** 演示5个核心场景，证明 AI 自动化运价管理的可行性和价值。

### 4.2 演示场景

| 场景 | 输入 | 演示效果 |
|------|------|---------|
| **① Excel 导入** | KMTC 运价表 (126行) | 上传 → 自动解析 → 展示结构化费率 → 一键入库 |
| **② 大型 Excel 导入** | NVO FAK (4000+行) | 批量解析 → 按 Sheet 分组展示 → 入库 |
| **③ 邮件文本解析** | 长荣报价邮件正文 | 粘贴文本 → AI 提取20+条航线费率 → 结构化展示 |
| **④ 截图识别** | WeChat 运价截图 | 上传图片 → AI Vision 识别 → 提取费率 |
| **⑤ 航线比价查询** | 输入 Shanghai → Busan | 展示所有供应商的报价对比 → 选择最优 |

### 4.4 技术风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| Claude API 调用延迟 | 截图/文本解析响应慢 | 异步处理 + 进度显示，前台不阻塞 |
| Excel 列结构变化 | 解析失败 | 预设多种模板 + 关键词模糊匹配 |
| PDF 表格提取不完整 | 费率数据丢失 | pdfplumber + Claude 双重校验 |
| 港口名不统一 | 数据无法关联 | 建立港口别名映射表（中/英/日/缩写） |

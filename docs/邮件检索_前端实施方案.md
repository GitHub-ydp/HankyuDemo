# 邮件智能检索 — 前端实施方案

> 本文档由业务分析师和方案架构师协作产出，经监督审核确认。Codex 按此文档实施。

---

## 一、项目背景

为阪急阪神开发"邮件智能检索"Demo，核心演示场景：

> 用户输入"从邮箱中检索和XX客户相关的往来邮件，按时间线进行概括汇总展示"
> → 系统从真实邮箱检索 → AI 按时间线分析汇总 → 结构化展示结果

**后端已完成**，本方案聚焦前端集成。

## 三、已有后端 API

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/v1/emails/index/imap?force=true&limit=200&since_date=2025-01-01` | 从 IMAP 拉取邮件并建索引 |
| POST | `/api/v1/emails/index/mock?force=true` | 用模拟数据建索引 |
| GET | `/api/v1/emails/search?q=xxx&top_k=10` | 语义搜索 |
| GET | `/api/v1/emails/analyze?q=xxx&top_k=15` | AI 分析（检索+时间线汇总） |

### API 响应格式

**注意**：后端 FastAPI 直接返回 JSON，经 Axios 拦截器后前端拿到的就是数据本身（无 code/data/message 包装）。

**search 返回**:
```json
{ "query": "xxx", "count": 10, "results": [
  { "id": "...", "subject": "...", "from": "...", "from_name": "...",
    "to": "...", "date": "2026-01-08T09:15:00+09:00", "score": 0.85,
    "content": "...", "category": "...", "tags": "...", "has_attachment": "True" }
]}
```

**analyze 返回**:
```json
{ "query": "xxx", "count": 15, "emails": [ ...同上... ],
  "analysis": "## 时间线汇总\n\n- **2026-01-08**..." }
```

**index 返回**:
```json
{ "status": "indexed", "count": 42 }
```

## 六、各文件详细设计

### 6.1 `frontend/src/types/email.ts`

```typescript
// 邮件搜索结果项
export interface EmailItem {
  id: string;
  subject: string;
  from: string;
  from_name: string;
  to: string;
  date: string;
  score: number;
  content: string;
  category: string;
  tags: string;
  has_attachment: string;
  folder: string;
}

// search API 响应
export interface EmailSearchResponse {
  query: string;
  count: number;
  results: EmailItem[];
}

// analyze API 响应
export interface EmailAnalyzeResponse {
  query: string;
  count: number;
  emails: EmailItem[];
  analysis: string; // Markdown
}

// index API 响应
export interface EmailIndexResponse {
  status: string;
  count: number;
}
```

### 6.2 `frontend/src/services/emailApi.ts`

```typescript
import api from './api';
import type { EmailSearchResponse, EmailAnalyzeResponse, EmailIndexResponse } from '../types/email';

export const emailApi = {
  // 从 IMAP 拉取并建索引
  indexFromImap: (params: { force?: boolean; limit?: number; since_date?: string }) =>
    api.post<unknown, EmailIndexResponse>('/emails/index/imap', null, { params, timeout: 120000 }),

  // 模拟数据建索引
  indexMock: (params?: { force?: boolean }) =>
    api.post<unknown, EmailIndexResponse>('/emails/index/mock', null, { params, timeout: 120000 }),

  // 语义搜索
  search: (q: string, top_k: number = 10) =>
    api.get<unknown, EmailSearchResponse>('/emails/search', { params: { q, top_k } }),

  // AI 分析
  analyze: (q: string, top_k: number = 15) =>
    api.get<unknown, EmailAnalyzeResponse>('/emails/analyze', { params: { q, top_k }, timeout: 60000 }),
};
```

**重要**：analyze 接口需要更长的 timeout（调 Claude API 可能 10~30 秒）。

### 6.7 `frontend/src/pages/EmailSearch.tsx`

主页面，整合所有组件。

**State**:
```typescript
const [indexCount, setIndexCount] = useState<number | null>(null);
const [loading, setLoading] = useState(false);
const [mode, setMode] = useState<'search' | 'analyze' | null>(null);
const [searchResults, setSearchResults] = useState<EmailItem[]>([]);
const [analyzeResult, setAnalyzeResult] = useState<{ analysis: string; emails: EmailItem[] } | null>(null);
const [currentQuery, setCurrentQuery] = useState('');
```

**逻辑**:
```typescript
const handleSearch = async (query: string) => {
  setLoading(true);
  setMode('search');
  setCurrentQuery(query);
  try {
    const res = await emailApi.search(query);
    setSearchResults(res.results);
    message.success(`找到 ${res.count} 封相关邮件`);
  } catch (e) {
    message.error(e instanceof Error ? e.message : '搜索失败');
  } finally {
    setLoading(false);
  }
};

const handleAnalyze = async (query: string) => {
  setLoading(true);
  setMode('analyze');
  setCurrentQuery(query);
  try {
    const res = await emailApi.analyze(query);
    setAnalyzeResult({ analysis: res.analysis, emails: res.emails });
    message.success('分析完成');
  } catch (e) {
    message.error(e instanceof Error ? e.message : '分析失败');
  } finally {
    setLoading(false);
  }
};
```

**JSX 结构**:
```tsx
<div style={{ display: 'flex', flexDirection: 'column', gap: 16 }}>
  <EmailConnection onIndexComplete={setIndexCount} />
  <EmailSearchInput
    onSearch={handleSearch}
    onAnalyze={handleAnalyze}
    loading={loading}
    disabled={indexCount === null || indexCount === 0}
  />
  {mode === 'search' && (
    <EmailResults results={searchResults} loading={loading} />
  )}
  {mode === 'analyze' && analyzeResult && (
    <EmailAnalysis
      analysis={analyzeResult.analysis}
      emails={analyzeResult.emails}
      loading={loading}
      query={currentQuery}
    />
  )}
</div>
```

## 八、关键交互规则

### 8.1 索引未完成时
- 搜索输入框 `disabled`
- 按钮灰色不可点击
- 显示提示"请先连接邮箱或加载模拟数据"

### 8.2 Loading 状态
- 索引中：按钮 loading + Spin + 文字提示
- 搜索中：结果区域 `Table loading`
- 分析中：显示三步骤文字动画（检索→分析→生成）

### 8.3 错误处理
- 所有 API 调用 try-catch，错误通过 `message.error` 展示
- 索引失败重置状态，允许重试
- 分析超时提示"分析耗时较长，请稍候"

### 8.4 相关度颜色
- score > 0.7 → 绿色 Tag
- score > 0.5 → 橙色 Tag
- score <= 0.5 → 灰色 Tag

### 8.5 快捷模板
点击模板 Tag 自动填入输入框，不自动触发搜索（让用户确认后再点按钮，方便演示讲解）。

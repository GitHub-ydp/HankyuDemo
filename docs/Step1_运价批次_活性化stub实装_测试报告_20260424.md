## 发现的问题

### P1-1（中）并发激活错误码 + SQL 原文泄漏
B1 场景，SQLite "database is locked" → 返回：
```
F-ACT-05: OperationalError: (sqlite3.OperationalError) database is locked
[SQL: UPDATE import_batches SET status=? ...][parameters: (...)]
```
Demo 风险低（无并发用户），但 UX 差，Demo 彩排前打磨。

建议：`activator.activate` 的 `except Exception` 对 `OperationalError` 单独分类（`F-ACT-07`），`detail` 只保留 `type(exc).__name__`，不 dump raw SQL。

### P1-2（中）dry_run on active 被拦成 already_active
`rate_batch_service.py:191-211` 对已 active 批次直接短路，不进入 dry_run 分支。业务意图是"dry_run 让营业心里有数"。UX 小瑕，不阻塞。

### P2-1（低）Ocean/NGB 路径零跑动
见上。

### P2-2（低）SQLite 下缺 partial unique index
`import_batches` 业务规则"同 file_type 同时刻最多 1 个 active"靠代码顺序保证，PG 迁移时并发可能撞车。建议：
```sql
CREATE UNIQUE INDEX uq_import_batch_one_active_per_file_type
ON import_batches (file_type) WHERE status = 'active';
```
SQLite→PG 时再做。

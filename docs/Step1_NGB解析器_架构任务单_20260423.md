## 3. 核心解析步骤（按业务需求 §3 + §4 翻译）

### 3.1 Sheet 路由

- `wb = load_workbook(path, data_only=True)`（NGB Lv.2/Lv.3 公式必须取计算结果，业务需求 §4.2 R-NGB-2）
- 遍历 `wb.sheetnames`，仅处理 `_RATE_SHEET == "Rate"`
- `sample` / `Shipping line name` 出现在 sheetnames 中也只跳过，不打 warning（业务需求 §10 BR-NGB-7 指导：业务正常的现象不报 warning）
- 若 `Rate` sheet 缺失 → warning `"Rate sheet missing in NGB workbook; nothing to parse"`，返回空 batch

### 3.2 Rate sheet 数据区扫描

- 范围：`for row_index in range(2, ws.max_row + 1)`（第 1 行是表头；业务需求 §3 已实证 ws.max_row=104）
- 提前终止条件：连续 ≥3 行整行空 → break（兜底极端原件）
- 每行：
  - `row = self._read_row_cells(ws, row_index)`（返回长度 55 list；索引 `row[0]` 对应 A 列，`row[i-1]` 对应第 i 列）
  - 若 `_is_empty_row(row)` → continue（业务需求 §4.6，41/42/67/83 行）
  - 调用 `_build_ngb_record(row, row_index, source_file=...)` 产出 0 或 1 条记录 + warnings

### 3.3 表头校验（不 fail-fast）

- 第一行读 A1/B1/E1/F1/I1/M1/N1 七个关键字段
- 期望值（业务需求 §3.1-3.3）：`Agent / P/S rate or Net Net / FCL/LCL / Shipping line / co-loader / Origin port / Place of Delivery (M) / Place of Delivery full (N)`
- 任一不匹配 → warning `"NGB Rate header mismatch at column {col}: expected '{expected_keyword}', got '{raw}'"`，**继续按固定列解析**（与 air.py:135 处理风格一致）
- 比较时统一 `_normalize_text + .lower() + 包含子串`，兼容换行 / 尾空格（业务需求 §3 多列表头有 `\n`、尾空格）

### 3.4 单行 → 单条 record 的核心逻辑（_build_ngb_record）

- **Step A 主标识读取**：
  - `agent = _normalize_text(row[_COL_AGENT - 1])`（实测全为 'HHE/NGB'）
  - `level_raw = _normalize_text(row[_COL_LEVEL - 1])`；若不在 `_RATE_LEVEL_VALUES` 三选一 → warning `"NGB row {r}: unrecognized rate_level '{raw}', skipped"` 并返回 None
  - `mode = _normalize_text(row[_COL_MODE - 1])`；若不在 {FCL, LCL} → warning + 跳过
  - `valid_from = _to_date(row[_COL_VALID_FROM - 1])` / `valid_to = _to_date(row[_COL_VALID_TO - 1])`
    - 任一为 None → warning `"NGB row {r}: cannot parse valid_from/valid_to from C/D"`，但不跳过该行（业务需求 §4.10 兜底取批次范围）
  - `shipping_line = _normalize_text(row[_COL_SHIPPING_LINE - 1])`（业务需求 §4.11：保留原文，不查字典）

- **Step B 起运/目的港**：
  - `origin_port_name = _normalize_text(row[_COL_ORIGIN_PORT - 1]) or _DEFAULT_ORIGIN_PORT`（业务需求 §4 "缺失起运港默认 NINGBO"）
  - 若用了默认值 → `extras["origin_source"] = "default_NINGBO"`
  - `destination_port_code = _normalize_text(row[_COL_PLACE_OF_DELIVERY_CODE - 1])`（M 列）
  - `destination_port_name = _normalize_text(row[_COL_PLACE_OF_DELIVERY_FULL - 1])`（N 列；业务需求 §10 BR-NGB-8：MOJ/MOJI 不归一）
  - `pod_code = _normalize_text(row[_COL_POD_CODE - 1])`（J 列实测多空，业务需求 §7 E-Q-NGB-5 默认 None）

- **Step C 主运费分流（业务需求 §4.7 R-NGB-7）**：
  - `mode == "FCL"`:
    - 读 `currency = _normalize_text(row[_COL_FCL_CURRENCY - 1]) or _DEFAULT_CURRENCY`（实测全 USD）
    - 读 R/S/T 三柜型；按 `level` 处理（详见 §3.5）
    - LCL 字段（AO/AP 等）**不读取**，避免 BR-NGB-2 误打 warning
  - `mode == "LCL"`:
    - 读 `currency = _normalize_text(row[_COL_LCL_CURRENCY - 1]) or _DEFAULT_CURRENCY`（实测 USD）
    - 读 AO/AP（按方/按吨）；按 `level` 处理
    - FCL 字段（R/S/T）不读取

- **Step D 附加费列（按字符串原文保留，业务需求 §4.8 R-NGB-8）**：
  - 所有附加费列统一走 `_extract_text_keep_newline` 或 `_normalize_text`，**不结构化**
  - FCL：U..AM 共 19 列；LCL：AQ..BB 共 12 列
  - 全部塞 `extras` 字典，键名规范见 §6.2

- **Step E 备注**：
  - `remarks = _extract_text_keep_newline(row[_COL_REMARKS - 1])`

- **Step F 写 record**：
  - 主字段映射见 §5 表格；其他全 None
  - `extras` 内容见 §6
  - `column_index_map` 内容见 §6.3（Lv.2/3 行的 R/S/T/AO/AP 不入此 map，业务需求 §11.1 + §10 AC-NGB-20）

### 3.5 公式列处理（业务需求 §4.2 R-NGB-2，P0 阻塞 E-Q-NGB-1 的实现）

- 业务出处：业务需求 §4.2 第 199-205 行；§7 E-Q-NGB-1 默认值 "允许 fallback，并打 warning；以原件 data_only 值为优先"
- 实现：`_to_decimal_or_none` 取到的值即 openpyxl `data_only=True` 模式下的"公式计算结果"
  - 若 `Lv.1` 行：直接取 R/S/T 数值；若 Lv.1 行也是 None → warning `"NGB row {r}: Lv.1 main rate is empty, cannot compute Lv.2/Lv.3 fallback"`，对应字段写 None
  - 若 `Lv.2 / Lv.3` 行：
    - 优先用 `_to_decimal_or_none(row[col-1])` 取 data_only 结果
    - 若取到 None（公式结果未缓存）→ 调 `_compute_lv_fallback` 现场计算
    - **fallback 条件**：必须能从**同一组的 Lv.1 行**取到对应列的数值
    - **难点**：当前行只能看到自己的 row，无法直接拿到 Lv.1。**实现策略**：
      - 在 `_parse_rate_sheet` 主循环中维护一个 `last_lv1_rates: dict[int, Decimal] | None`（key=列号，value=Lv.1 数值），每遇到 Lv.1 行就重置为该行的 R/S/T (FCL) 或 AO/AP (LCL) 实测值
      - 处理 Lv.2/Lv.3 行时，传入 `last_lv1_rates` 给 `_build_ngb_record`；若 data_only 取到 None 且 `last_lv1_rates[col]` 有值 → 调 `_compute_lv_fallback(last_lv1_rates[col], level)`
      - fallback 触发时打 warning `"NGB row {r}: {level} {col_name} formula not cached, fallback computed = {value}"`
- `_compute_lv_fallback` 公式（业务需求 §0 第 33 行明示）：
  - Lv.2: `ROUNDUP(lv1 * Decimal('1.1'), -1)` → 即向上取整到 10 的倍数；用 `math.ceil(lv1 * 1.1 / 10) * 10` 实现
  - Lv.3: `ROUNDUP(lv1 * Decimal('1.2'), -1)` → 同上 ×1.2
  - 返回 `Decimal`；输入 None → 返回 None
- 注意：fallback 仅作用于 R/S/T/AO/AP 主运费列；其他附加费列（V/W/.../BA）取 None 就 None，不做 fallback（业务需求 §4.4 R-NGB-4 已说明 Lv.2/3 非运费列是 `=A2` 公式，data_only 应当恒能取到值；若取不到也按 None 处理，不做猜测）

### 3.6 Shipping line name sheet（业务需求 §3.3、§5.3）

- **不解析、不入主表**
- 不读取，不打 warning，不在 metadata 暴露
- 监工抽查：在 `parse` 主入口看 sheetnames 时，仅对 `_RATE_SHEET` 调 `_parse_rate_sheet`，其他 sheet 直接 skip

### 3.7 sample sheet（业务需求 §2、§5.3）

- 同 3.6 处置：跳过、不打 warning
- 业务需求 §10 BR-NGB-5 警告：禁止用 sample 列布局来匹配 Rate（17 列 vs 55 列错位会毁掉整批）

## 4. ParsedRateBatch 元信息

- `file_type = Step1FileType.ocean_ngb`
- `source_file = path.name`
- `adapter_key = "ocean_ngb"`
- `effective_from / effective_to`（业务需求 §4.10 R-NGB-10）：
  - 取**所有 Lv.1 行**的 (C, D) 集合
  - 若集合 size == 1 → 直接用
  - 若 size > 1 → `min(from)` / `max(to)` 兜底，加 warning `"NGB workbook has {n} distinct (effective_from, effective_to) tuples; batch range fallback to min(from) / max(to)"`（业务需求 §7 E-Q-NGB-2 默认）
  - 若全空 → batch 级 None + warning
- `metadata`：
  - `parser_version = "ocean_ngb_v1"`
  - `ngb_origin_assumption = "default origin = NINGBO when I column is empty"`
  - `formula_fallback_count = N`（fallback 触发次数；用于 Demo 当天可见性）
  - `sheets = [{"sheet_name": "Rate", "total_rows": <len(records)>, "effective_from": ..., "effective_to": ...}]`
  - `record_kind_distribution = {"ocean_ngb_fcl": N, "ocean_ngb_lcl": M}`
- `warnings`：经 `_dedupe_warnings` 去重

### 6.3 column_index_map（业务需求 §11.1 关键约定）

- 用途：Writer 端按 `(row_index, col_index, value)` 三元组逐 cell 回填模板（writer/ocean_ngb.py:51-57）
- 类型：`dict[int, Any]`，key=1-based 列号，value=要写入的值
- **必须遵守 §11.1 规则**：
  - **Lv.1 行**：把所有"原文非空、非公式"的字段放进 map（包括 R/S/T/AO/AP 主运费数值）
  - **Lv.2 / Lv.3 行**：**不要**把 R/S/T/AO/AP 这五个公式 cell 写入 map（让 Writer 端 `safe_set` 守卫保留原模板的 `=ROUNDUP(...)` 公式自动联动；业务需求 §11.1 + §10 AC-NGB-20）
  - **Lv.2 / Lv.3 行的"非运费列继承公式"**（如 `A3='=A2'`）也**不要**写入 map（同理保留公式联动）
  - **更稳妥的实现**：本任务建议 Writer 端的 `safe_set` 已经有 `is_formula_cell` 守卫（writers/base.py:15-20，38-39），即使 parser 误传公式 cell 的值也会被拒绝写入。但 parser 仍按 §11.1 规则**主动不传**，理由：减少 Writer 端日志噪音 + 让 column_index_map 真实反映"parser 想覆盖什么"
- **建议实现**（在 `_build_column_index_map` 内）：
  - 接收参数：`row, row_index, level, mode`
  - 输出：`dict[int, Any]`
  - 逻辑：
    - Lv.1 行：把 row 中所有非 None、非空字符串的 cell 放进 map（key=列号）
    - Lv.2/Lv.3 行：跳过 `_FCL_RATE_COLS`（R/S/T）和 `_LCL_RATE_COLS`（AO/AP）；其他列**也建议跳过**（因为大概率是 `=<lv1>` 公式，但 data_only 取到的是计算值；写回去会覆盖公式）
    - **简化策略（推荐）**：Lv.2/Lv.3 行的 column_index_map 直接返回 `{}`（空 dict）。Writer 端遇到空 map 不写任何 cell，模板原公式全保留。这与业务需求 §11.1 一致
